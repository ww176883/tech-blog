# 如何准备系统设计面试？

## 📌 题目分析

系统设计面试是后端工程师面试中**最重要**的环节之一，主要考察：

- 系统架构设计能力
- 技术选型和权衡能力
- 解决复杂问题的思路
- 沟通表达能力

<!-- more -->

## 🎯 核心考察点

### 1. 需求分析能力（5 分钟）

**面试官期望：**
- 能主动澄清需求边界
- 能识别核心功能和次要功能
- 能估算系统规模和性能指标

**准备方法：**
```
准备一套标准问题模板：
1. 用户规模多大？（DAU/MAU）
2. 主要功能有哪些？优先级？
3. 性能要求？（QPS、延迟、可用性）
4. 数据规模？（存储量、增长量）
```

### 2. 高层设计能力（10 分钟）

**面试官期望：**
- 能画出清晰的系统架构图
- 组件划分合理
- 数据流向清晰

**准备方法：**
```
掌握标准架构组件：
- 负载均衡器（Nginx/HAProxy）
- API 网关
- 应用服务层
- 缓存层（Redis/Memcached）
- 数据库层（MySQL/PostgreSQL）
- 消息队列（Kafka/RabbitMQ）
- 对象存储（S3/OSS）
```

### 3. 详细设计能力（20 分钟）

**面试官期望：**
- 能深入关键组件的设计细节
- 能处理边界情况和异常
- 能进行技术选型和权衡

**准备方法：**
```
重点掌握：
1. 数据库表设计
2. 缓存策略
3. 分库分表方案
4. 一致性保证
5. 容错和降级
```

### 4. 扩展和优化（10 分钟）

**面试官期望：**
- 能识别系统瓶颈
- 能提出优化方案
- 能评估方案成本

## 📚 知识体系准备

### 基础组件（必须掌握）

| 组件 | 核心知识点 | 高频问题 |
|------|----------|---------|
| **负载均衡** | 轮询、加权、一致性哈希 | 如何保证会话粘性？ |
| **缓存** | 缓存策略、穿透/击穿/雪崩 | 如何保证缓存一致性？ |
| **数据库** | 索引、事务、锁 | 如何优化慢查询？ |
| **消息队列** | 异步解耦、削峰填谷 | 如何保证消息不丢失？ |
| **分布式 ID** | 雪花算法、号段模式 | 如何保证 ID 全局唯一？ |

### 高级主题（加分项）

| 主题 | 核心知识点 |
|------|----------|
| **分布式事务** | 2PC、TCC、最终一致性 |
| **服务治理** | 熔断、降级、限流 |
| **监控告警** | 指标采集、日志聚合 |
| **容灾备份** | 多活、异地容灾 |

## 💡 面试技巧

### 1. 主动沟通

```
❌ 错误做法：拿到题目直接开始设计
✅ 正确做法：先问清楚需求和约束条件
```

### 2. 由简到繁

```
第一步：设计一个最小可用系统
第二步：识别瓶颈和问题
第三步：逐步优化和扩展
```

### 3. 善用画图

```
推荐工具：
- 白板/纸笔（线下面试）
- Excalidraw/Draw.io（在线面试）

画图要点：
1. 先画整体架构
2. 再画数据流向
3. 标注关键组件
```

### 4. 量化指标

```
❌ 模糊表述："系统要能抗住高并发"
✅ 量化表述："系统需要支持 10 万 QPS，P99 延迟<100ms"
```
