# 如何设计一个短链系统？⭐

## 📌 题目分析

**短链系统功能：**
- 将长 URL 转换为短 URL（如：https://t.cn/A1B2C3D）
- 用户访问短链时重定向到原长链
- 统计访问数据（PV、UV、来源等）

**核心挑战：**
1. 如何生成唯一的短链 ID？
2. 如何保证高性能读写？
3. 如何防止恶意攻击？
4. 如何统计访问数据？

## 💡 核心解决方案

### 1. 短链 ID 生成方案

#### 方案一：分布式 ID + Base62 编码（推荐）

```java
// 1. 使用雪花算法生成分布式 ID
long id = snowflake.nextId(); // 如：1234567890123456789

// 2. Base62 编码（0-9, a-z, A-Z）
String shortKey = base62Encode(id); // 如："Ab3Cd4"

// Base62 编码实现
private static final String BASE62 = "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ";

public String base62Encode(long num) {
    StringBuilder sb = new StringBuilder();
    while (num > 0) {
        sb.append(BASE62.charAt((int)(num % 62)));
        num /= 62;
    }
    return sb.reverse().toString();
}
```

**优点：**
- 性能高，无数据库依赖
- ID 有序，便于数据库索引
- 62 进制，6 位可表示 62^6 ≈ 568 亿个短链

**缺点：**
- 需要保证 ID 生成器高可用

#### 方案二：数据库自增 ID

```java
// 使用数据库自增 ID
@Transactional
public String createShortUrl(String longUrl) {
    // 1. 插入记录，获取自增 ID
    UrlMapping mapping = new UrlMapping();
    mapping.setLongUrl(longUrl);
    urlMapper.insert(mapping); // 自增 ID: 123456
    
    // 2. Base62 编码
    String shortKey = base62Encode(mapping.getId()); // "1Abc"
    
    return shortKey;
}
```

**优点：**
- 实现简单
- ID 全局唯一

**缺点：**
- 数据库压力大
- 单点瓶颈

#### 方案三：MD5/SHA 哈希

```java
// 对长链进行哈希
String hash = md5(longUrl); // 32 位
String shortKey = hash.substring(0, 6); // 取前 6 位
```

**优点：**
- 相同长链生成相同短链

**缺点：**
- 存在哈希冲突
- 需要冲突检测机制

### 2. 重定向方案

#### HTTP 301（永久重定向）vs 302（临时重定向）

| 类型 | 特点 | 使用场景 |
|------|------|---------|
| **301** | 浏览器缓存，后续直接访问长链 | SEO 优化、永久短链 |
| **302** | 每次访问短链服务器 | 需要统计访问数据 |

**推荐：使用 302**（便于统计）

```java
@GetMapping("/{shortKey}")
public void redirect(@PathVariable String shortKey, 
                     HttpServletResponse response,
                     HttpServletRequest request) throws IOException {
    // 1. 查询缓存
    String longUrl = redisTemplate.opsForValue().get("url:" + shortKey);
    
    // 2. 缓存未命中，查数据库
    if (longUrl == null) {
        UrlMapping mapping = urlMapper.findByShortKey(shortKey);
        if (mapping == null) {
            response.sendError(404);
            return;
        }
        longUrl = mapping.getLongUrl();
        // 写入缓存（1 天过期）
        redisTemplate.opsForValue().set("url:" + shortKey, longUrl, 1, TimeUnit.DAYS);
    }
    
    // 3. 异步发送访问日志
    AccessLog log = new AccessLog(shortKey, request, new Date());
    kafkaTemplate.send("access-log", log);
    
    // 4. 302 重定向
    response.setStatus(302);
    response.setHeader("Location", longUrl);
}
```

CREATE TABLE access_log_00 (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    short_key VARCHAR(10) NOT NULL,
    user_id VARCHAR(64), -- 用户标识
    ip VARCHAR(45), -- IPv6
    referer VARCHAR(500), -- 来源
    user_agent TEXT, -- 浏览器
    country VARCHAR(50), -- 国家
    province VARCHAR(50), -- 省份
    city VARCHAR(50), -- 城市
    access_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_short_key_time (short_key, access_time)
) PARTITION BY RANGE (YEAR(access_time)) (
    PARTITION p2025 VALUES LESS THAN (2026),
    PARTITION p2026 VALUES LESS THAN (2027)
);

## 📈 统计功能设计

### 1. 实时统计（Redis）

```java
// 日 PV
redisTemplate.opsForHyperLogLog().add("stats:pv:" + shortKey + ":" + date, userId);

// 独立 UV
redisTemplate.opsForSet().add("stats:uv:" + shortKey + ":" + date, deviceId);

// 来源统计
redisTemplate.opsForHash().increment("stats:referer:" + shortKey + ":" + date, refererDomain, 1);

// 地域统计
redisTemplate.opsForHash().increment("stats:city:" + shortKey + ":" + date, city, 1);
```

### 2. 离线分析（ClickHouse）

```sql
-- 创建统计视图
CREATE MATERIALIZED VIEW daily_stats ENGINE = SummingMergeTree()
ORDER BY (short_key, date)
AS SELECT 
    short_key,
    toDate(access_time) as date,
    count() as pv,
    uniq(user_id) as uv,
    uniq(ip) as unique_ip
FROM access_log
GROUP BY short_key, date;

-- 查询今日数据
SELECT * FROM daily_stats 
WHERE short_key = 'Ab3Cd4' AND date = today();
```

## 🔧 完整流程

### 短链生成流程

```
1. 用户提交长 URL
   ↓
2. 校验 URL 合法性（格式、白名单）
   ↓
3. 检查是否已存在（MD5 查重）
   ↓ (已存在)
4. 返回已有短链
   ↓ (不存在)
5. 生成短链 ID（雪花算法）
   ↓
6. Base62 编码
   ↓
7. 写入数据库
   ↓
8. 返回短链
```

### 重定向流程

```
1. 用户访问短链
   ↓
2. 查询 Redis 缓存
   ↓ (未命中)
3. 查询数据库
   ↓
4. 写入 Redis 缓存
   ↓
5. 异步记录访问日志
   ↓
6. 302 重定向到长链
```

## ⚠️ 常见问题

### Q1: 短链冲突怎么办？

**方案：**
1. 数据库唯一索引保证
2. 冲突时重新生成 ID

```java
while (true) {
    try {
        urlMapper.insert(mapping);
        break;
    } catch (DuplicateKeyException e) {
        // 生成新 ID 重试
        mapping.setShortKey(base62Encode(snowflake.nextId()));
    }
}
```

### Q2: 缓存穿透怎么办？

**方案：**
1. 布隆过滤器预判
2. 空值也缓存（短过期时间）

```java
// 布隆过滤器检查
if (!bloomFilter.mightContain(shortKey)) {
    return Result.fail("短链不存在");
}

// 空值缓存
if (longUrl == null) {
    redisTemplate.opsForValue().set("url:" + shortKey, "", 5, TimeUnit.MINUTES);
    return null;
}
```

### Q3: 海量数据存储？

**方案：**
1. 访问日志分表（按时间/短链哈希）
2. 冷热数据分离
3. 定期归档历史数据
